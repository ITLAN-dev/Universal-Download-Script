# Universal Download Script

Скрипт для загрузки файлов с поддержкой докачки, проверки целостности и тремя режимами работы.

## ✨ Возможности

- **Три режима загрузки**:
  - По паттерну (patch0.tar.gz ... patchN.tar.gz)
  - Из файла со списком URL
  - Одиночный файл
  
- **Умная проверка файлов**:
  - Определяет полностью загруженные
  - Находит частично загруженные (докачка)
  - Выявляет поврежденные (перезагрузка)
  
- **Поддержка докачки** (resume):
  - curl (флаг -C -)
  - wget (флаг -c)
  - aria2c (--continue=true)
  
- **Автоопределение доступных загрузчиков**
- **Цветной вывод с эмодзи** для лучшей читаемости
- **Прогресс-бар** для каждого файла
- **Проверка размера** на сервере без скачивания

## 🔧 Требования

Обязательно:
- Bash 4.0+
- Один из загрузчиков:
  - `curl` (рекомендуется)
  - `wget`
  - `aria2c`

Опционально:
- `bc` - для форматирования размеров (будет использовать встроенные средства если отсутствует)

## 📦 Установка

```bash
# Скачать скрипт
wget -O download.sh https://raw.githubusercontent.com/ITLAN-dev/Universal-Download-Script/master/downloader.sh

# Или curl
curl -L -o download.sh https://raw.githubusercontent.com/ITLAN-dev/Universal-Download-Script/master/downloader.sh

# Сделать исполняемым
chmod +x download.sh
```

## 🚀 Режимы работы

### 1. Режим паттерна (Pattern Mode)
Для загрузки последовательно именованных файлов:
```bash
./download.sh "URL_PREFIX" START END [OUTPUT_DIR] [EXTENSION] [MIN_SIZE]
```

### 2. Режим из файла (File Mode)
Для загрузки из списка URL:
```bash
./download.sh -f FILE_WITH_URLS [OUTPUT_DIR] [MIN_SIZE]
```

### 3. Режим одного URL (Single URL Mode)
Для загрузки одного файла:
```bash
./download.sh -u SINGLE_URL [OUTPUT_DIR] [MIN_SIZE]
```

## ⚙️ Принцип работы

### Алгоритм проверки файла
```
1. Файл существует?
   ├─ Нет → начинаем загрузку
   └─ Да → проверяем размер
        ├─ Меньше MIN_SIZE → поврежден → удаляем → загружаем
        └─ Больше MIN_SIZE → сравниваем с сервером
             ├─ Равны → ✅ готово
             ├─ Меньше → частично → докачиваем
             └─ Больше → ошибка → удаляем → загружаем
```

### Процесс загрузки
1. Получение размера файла на сервере (HEAD-запрос)
2. Проверка существующего локального файла
3. Выбор стратегии (новая загрузка / докачка)
4. Попытка загрузки доступными инструментами:
   - Сначала curl (с прогрессом)
   - Затем wget (если curl недоступен)
   - Наконец aria2c (если есть)
5. Повторные попытки при ошибках (до 5 раз)
6. Финальная проверка целостности

## 📝 Примеры

### Базовые примеры

```bash
# Скачать один файл
./download.sh -u "https://example.com/file.zip" ./

# Скачать файлы patch0.tar.gz ... patch50.tar.gz
./download.sh "https://example.com/files/patch" 0 50 ./downloads

# Скачать с другим расширением
./download.sh "https://example.com/data/part" 1 10 ./data zip
```

### Продвинутое использование

```bash
# Минимальный размер 10MB для проверки целостности
./download.sh -u "https://example.com/bigfile.iso" ./iso 10485760

# Загрузка из файла с комментариями (urls.txt)
# http://site.com/file1.zip
# http://site.com/file2.tar.gz
# # это комментарий
./download.sh -f urls.txt ./downloads

# Паттерн с кастомной директорией и расширением
./download.sh "https://cdn.example.com/video/part" 1 20 ./videos mp4
```

### Пример файла с URL (urls.txt)
```text
# Видео файлы
https://example.com/videos/intro.mp4
https://example.com/videos/chapter1.mp4
https://example.com/videos/chapter2.mp4

# Архивы
https://backup.example.com/data/backup-2024-01.tar.gz
https://backup.example.com/data/backup-2024-02.tar.gz

# Документация
https://docs.example.com/manual.pdf
```

## 📊 Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `URL_PREFIX` | Префикс URL для паттерна | обязательный |
| `START` | Начальный индекс | обязательный |
| `END` | Конечный индекс | обязательный |
| `OUTPUT_DIR` | Директория сохранения | `./downloads` |
| `EXTENSION` | Расширение файлов | `tar.gz` |
| `MIN_SIZE` | Минимальный размер (байт) | `1000000` (1MB) |
| `-f FILE` | Режим из файла | - |
| `-u URL` | Режим одного URL | - |
| `-h` | Показать помощь | - |

## 🔄 Статусы файлов

| Статус | Описание | Действие |
|--------|----------|----------|
| `complete` | Полностью загружен | Пропуск |
| `partial` | Частично загружен | Докачка |
| `corrupt` | Слишком маленький | Удаление + новая загрузка |
| `invalid` | Больше чем на сервере | Удаление + новая загрузка |
| `missing` | Файл отсутствует | Новая загрузка |
| `valid` | Нет данных с сервера | Считаем OK |
